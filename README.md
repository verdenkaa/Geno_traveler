## О проекте

Проект демонстрирует эффективное применение генетического алгоритма для решения классической NP-трудной задачи — задачи коммивояжёра (Traveling Salesman Problem). Программа находит приближённо-оптимальный маршрут, проходящий через все заданные города и возвращающийся в начальный, минимизируя общее расстояние.

**Особенности**:
- Гибкая настройка всех параметров генетического алгоритма.
- Три способа ввода координат городов: вручную, из текстового файла, автоматическая генерация случайных городов.
- Сохранение истории лучших маршрутов в JSON-файл (с возможностью задать шаг сохранения).
- Визуализация процесса оптимизации на Python (matplotlib): анимация изменения маршрута и график сходимости.

---

## Решаемая задача

Дано `N` городов с координатами `(x, y)`. Найти замкнутый маршрут (цикл), проходящий через каждый город ровно один раз и возвращающийся в стартовый город, с минимальной суммарной длиной пути.

---

## Генетический алгоритм

| Компонент | Реализация |
|-----------------------|----------------------------------------------------------------------------|
| Представление решения | Перестановка городов (список целых чисел, замкнутый первым и последним) |
| Начальная популяция | Смесь случайных и жадных маршрутов (управляется параметром `greedyProbability`) |
| Селекция (отбор) | Рабочая группа случайного размера, два лучших в группе – родители |
| Кроссовер | Порядковый кроссовер (Order Crossover, OX) |
| Мутация | Обмен двух случайных городов с вероятностью `mutationChance` |
| Замена | Худший маршрут в рабочей группе заменяется потомком |

## Параметры алгоритма
| Параметр | Значение по умолчанию (пример) | Описание |
|----------|--------------------------------|----------|
| populationSize |	100–500 |	Количество маршрутов в популяции. |
| maxGenerations |	500–2000 |	Число поколений. |
| wrGroupSize	| 10–20% от populationSize |	Размер рабочей группы для отбора. |
| mutationChance |	0.1–0.3 |	Вероятность мутации (обмена двух городов). |
| greedyNeighbors |	3–10 |	Сколько ближайших соседей использовать при жадном старте. |
| greedyProbability |	50–80 |	Шанс создания жадного маршрута (остальные – случайные) в начальной популяции. |

## Визуализация
После получения JSON-файла скрипт visualizer.py покажет интерактивную анимацию:
- Слева – карта городов и лучший маршрут для текущего поколения.
- Справа – график сходимости (длина лучшего маршрута от поколения).
